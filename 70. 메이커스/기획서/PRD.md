# Product Requirements Document (PRD)

## 1. 개요

**제품명**: TaskFlow - 팀 협업 작업 관리 도구  
**버전**: 1.0  
**작성일**: 2026년 2월 6일  
**작성자**: 제품팀  
**상태**: Draft

## 2. 문제 정의

**현재 상황**  
현재 중소형 스타트업 팀들은 Jira는 너무 복잡하고, Trello는 기능이 부족하다고 느낍니다. 특히 10-50명 규모의 팀에서 간단하면서도 충분한 기능을 가진 작업 관리 도구가 필요합니다.

**타겟 사용자**

- 10-50명 규모의 스타트업 팀
- 개발자, 디자이너, PM이 혼재된 크로스펑셔널 팀
- 주로 원격 근무 또는 하이브리드 근무 환경

**핵심 문제**

- 기존 도구들의 복잡한 설정과 학습 곡선
- 팀원 간 작업 가시성 부족
- 우선순위 설정의 어려움

## 3. 목표 및 성공 지표

**비즈니스 목표**

- 출시 후 6개월 내 1,000개 팀 온보딩
- 월간 활성 사용자(MAU) 5,000명 달성
- 유료 전환율 15% 달성

**사용자 목표**

- 새 팀원이 5분 내에 첫 작업 생성
- 일일 작업 완료율 30% 향상
- 팀 커뮤니케이션 시간 20% 감소

**성공 지표 (KPIs)**

- 주간 활성 사용자 수 (WAU)
- 작업 완료율
- 평균 응답 시간
- NPS 점수 > 40

## 4. 핵심 기능 (Features)

### 4.1 MVP 기능 (P0 - Must Have)

**4.1.1 작업 생성 및 관리**

- 작업 제목, 설명, 담당자, 마감일 설정
- 작업 상태: To Do, In Progress, Done
- 우선순위: High, Medium, Low
- 파일 첨부 (최대 10MB)

**4.1.2 보드 뷰**

- 칸반 스타일 보드
- 드래그 앤 드롭으로 상태 변경
- 담당자별 필터링
- 우선순위별 정렬

**4.1.3 팀 협업**

- 작업 댓글 기능
- @멘션으로 팀원 태그
- 실시간 알림 (인앱, 이메일)

**4.1.4 사용자 관리**

- 이메일 기반 회원가입/로그인
- 팀 초대 (이메일 링크)
- 역할: Admin, Member

### 4.2 Phase 2 기능 (P1 - Should Have)

- 리스트 뷰 추가
- 작업 검색 기능
- 마일스톤 설정
- 작업 템플릿

### 4.3 향후 고려 기능 (P2 - Nice to Have)

- 타임 트래킹
- 간트 차트 뷰
- API 연동
- Slack/Discord 통합

## 5. 사용자 스토리

**US-001**: 팀 리더로서, 새 프로젝트를 위한 보드를 생성하고 팀원들을 초대할 수 있어야 한다.

**US-002**: 개발자로서, 내게 할당된 작업들을 한눈에 보고 우선순위에 따라 처리할 수 있어야 한다.

**US-003**: PM으로서, 각 작업의 진행 상태를 실시간으로 확인하고 병목 지점을 파악할 수 있어야 한다.

**US-004**: 디자이너로서, 작업에 디자인 파일을 첨부하고 피드백을 댓글로 받을 수 있어야 한다.

## 6. 사용자 플로우

```
1. 회원가입/로그인
   ↓
2. 팀 생성 또는 팀 초대 수락
   ↓
3. 보드 생성
   ↓
4. 작업 생성
   ↓
5. 작업 할당 및 상태 관리
   ↓
6. 협업 (댓글, 멘션)
```

## 7. 기술 요구사항

**프론트엔드**

- React 18+
- TypeScript
- TailwindCSS
- 반응형 디자인 (모바일 지원)

**백엔드**

- Node.js + Express
- PostgreSQL
- Redis (캐싱)
- WebSocket (실시간 업데이트)

**인프라**

- AWS EC2/RDS
- S3 (파일 저장)
- CloudFront (CDN)

**성능 기준**

- 페이지 로드 시간 < 2초
- API 응답 시간 < 200ms
- 99.9% 업타임

## 8. 디자인 요구사항

**원칙**

- 미니멀하고 직관적인 UI
- 키보드 단축키 지원
- 다크 모드 지원

**접근성**

- WCAG 2.1 AA 준수
- 스크린 리더 지원
- 키보드 네비게이션

## 9. 제약사항 및 리스크

**제약사항**

- 개발 기간: 3개월
- 개발 인력: 2명 (풀스택)
- 초기 예산: 제한적

**리스크**

- 경쟁 제품 대비 차별화 부족 → 독특한 UX로 해결
- 확장성 이슈 → 초기부터 아키텍처 고려
- 사용자 획득 비용 → 무료 티어로 진입장벽 낮춤
