## 1. 배경 및 문제 정의

### 1.1 현재 상황

원티드의 2025년 4분기 데이터를 보면:

- 평균 공고당 지원자 수: 약 127명
- 채용 담당자가 서류 검토에 쓰는 시간: 지원자당 평균 3-5분
- 서류 합격률: 약 15-20%
- **문제점**: 서류만으로는 지원자의 커뮤니케이션 스킬, 태도, 실제 역량을 파악하기 어려움

### 1.2 사용자 인터뷰 인사이트

**채용 담당자 인터뷰 (5명)**

- "이력서는 좋은데 막상 면접 보면 안 맞는 경우가 많아요"
- "1차 면접까지 가는데 시간이 너무 오래 걸려요"
- "지원자가 우리 회사에 진심인지 알고 싶어요"

**지원자 인터뷰 (8명)**

- "면접이 처음이라 너무 떨려요. 연습하고 싶은데..."
- "어떤 질문이 나올지 미리 알면 좋겠어요"
- "면접 실력을 키우고 싶은데 기회가 없어요"

### 1.3 타겟 사용자

**Primary**:

- 스타트업/중소기업 채용담당자 (HR 전담 인력 1-2명)
- 연 50-200명 채용하는 규모

**Secondary**:

- 경력 1-3년차 이직 준비자
- 신입 구직자

## 3. 핵심 기능 명세

### 3.1 MVP 기능 (Phase 1 - Must Have)

#### 기능 1: AI 면접관 설정 (채용담당자)

**설명**: 채용담당자가 포지션별로 면접 질문을 설정

**상세 요구사항**:

- [ ] 포지션 선택 (개발자, 디자이너, 마케터 등 - 직군 드롭다운)
- [ ] 질문 개수 선택 (3개 / 5개 / 7개)
- [ ] 질문 타입 선택
    - 기본 질문 (자기소개, 지원동기)
    - 직무 질문 (AI가 직군별로 추천)
    - 커스텀 질문 (직접 입력 - 최대 3개)
- [ ] 제한 시간 설정 (질문당 1-3분)
- [ ] 재시도 허용 여부 (1회 / 무제한)

**화면 구성**:

```
[포지션 설정]
- 직군: [드롭다운]
- 경력: 신입 / 경력 [토글]

[질문 구성]
질문 1: 간단히 자기소개를 해주세요 (기본, 삭제 불가)
질문 2: [AI 추천 질문 보기] 또는 [직접 입력]
질문 3: ...
[+ 질문 추가하기] (최대 7개)

[면접 설정]
- 답변 시간: [1분 / 2분 / 3분] 선택
- 재시도: [1회만 / 무제한] 
- 마감일: [날짜 선택]

[저장하기] [미리보기]
```

#### 기능 2: 사전면접 응시 (지원자)

**진입 경로**:

```
공고 상세페이지 → 지원하기 버튼 클릭 
→ (기존) 이력서 제출 
→ (NEW) "사전면접 보기" 버튼 노출 (선택사항)
```

**면접 플로우**:

1. 안내 화면
    - 질문 개수, 예상 소요 시간 안내
    - 카메라/마이크 권한 요청
    - 테스트 녹화 (5초 - "안녕하세요" 말해보기)
2. 면접 진행
    - 질문 1개씩 노출
    - 준비 시간 10초 → 자동으로 녹화 시작
    - 제한 시간 타이머 표시
    - 답변 완료 후 → 다음 질문
3. 완료 화면
    - "제출하시겠습니까?"
    - 제출 시 수정 불가 안내
    - (optional) 재시도 가능시 "다시보기" 버튼

**기술 요구사항**:

- WebRTC 기반 녹화
- 영상 포맷: MP4, 최대 화질 720p
- 파일 크기: 질문당 최대 50MB
- 모바일 웹 지원 필수

#### 기능 3: 면접 영상 리뷰 (채용담당자)

**화면 구성**:

```
[지원자 목록]
├─ 사전면접 완료 (32명) 👈 새 탭
├─ 사전면접 미완료 (85명)
└─ 전체 지원자 (117명)

[지원자 카드]
- 이름, 경력, 지원일
- [사전면접 보기] 버튼
- 태그: "면접 완료 ✓" "미완료"
```

**영상 뷰어**:

- 질문별 탭 구성
- 배속 재생 (1x, 1.5x, 2x)
- 메모 기능 (지원자에게는 비공개)
- 평가 버튼: 👍 관심있음 / 👎 패스 / ⭐ 북마크

**AI 인사이트 (Phase 2에서 추가 예정)**:

- 답변 키워드 분석
- 말하기 속도, 명확성 점수
- 감정 분석 (긍정/자신감)

#### 기능 4: 알림 시스템

**지원자**:

- 공고 지원 시: "사전면접으로 합격 확률을 높여보세요" (인앱 + 푸시)
- 면접 완료 시: "면접 제출 완료! 곧 결과를 받으실 수 있어요"

**채용담당자**:

- 지원자가 사전면접 완료 시: "새로운 사전면접 영상이 도착했어요" (이메일 + 인앱)

## 4. 사용자 스토리

### 채용담당자 관점

```
US-01: 채용담당자로서, 개발자 포지션에 맞는 기술 질문을 빠르게 설정하고 싶다.
  - Acceptance Criteria:
    ✓ 직군 선택 시 AI가 추천 질문 5개 이상 제시
    ✓ 클릭 한 번으로 질문 추가 가능
    ✓ 설정 완료까지 5분 이내

US-02: 채용담당자로서, 출퇴근 시간에 모바일로 면접 영상을 빠르게 확인하고 싶다.
  - Acceptance Criteria:
    ✓ 모바일 웹에서 영상 재생 가능
    ✓ 2배속 재생 지원
    ✓ 북마크/패스 버튼 터치 간편

US-03: 채용담당자로서, 특정 질문에 대한 답변만 빠르게 비교하고 싶다.
  - Acceptance Criteria:
    ✓ 질문별 탭으로 바로 이동
    ✓ 지원자 2-3명 선택해서 같은 질문 답변 비교
```

### 지원자 관점

```
US-04: 지원자로서, 첫 면접이라 긴장되는데 미리 연습하고 싶다.
  - Acceptance Criteria:
    ✓ 테스트 녹화로 내 모습 확인 가능
    ✓ 안내 문구가 친절하고 명확함
    ✓ 재시도 가능하면 안심됨

US-05: 지원자로서, 이동 중에도 모바일로 간편하게 면접 보고 싶다.
  - Acceptance Criteria:
    ✓ 모바일 브라우저에서 카메라 작동
    ✓ 데이터 사용량 합리적 (200MB 이하)
    ✓ 중간에 끊겨도 이어서 가능

US-06: 지원자로서, 내 답변이 괜찮았는지 피드백을 받고 싶다. (Phase 2)
```

## 7. 기술 스택 및 요구사항

### 7.1 프론트엔드

- React 18 (기존 원티드 웹과 동일)
- WebRTC API (getUserMedia)
- MediaRecorder API
- 반응형 필수 (모바일 최적화)

### 7.2 백엔드

- Node.js + Express (기존 스택 유지)
- API 엔드포인트:
    - `POST /api/pre-interview/create` - 면접 설정 생성
    - `POST /api/pre-interview/upload` - 영상 업로드
    - `GET /api/pre-interview/:id` - 영상 조회
    - `POST /api/pre-interview/evaluate` - 평가 저장

### 7.3 인프라

- AWS S3: 영상 파일 저장
- CloudFront: CDN으로 스트리밍
- RDS (PostgreSQL): 메타데이터 저장
- ElastiCache (Redis): 영상 업로드 상태 캐싱

### 7.4 AI/ML (Phase 2)

- OpenAI Whisper API: STT (음성→텍스트)
- GPT-4: 답변 키워드 분석, 피드백 생성
- (검토 필요) 감정 분석 API

### 7.5 성능 요구사항

- 영상 업로드 속도: 1분 영상 기준 10초 이내
- 스트리밍 버퍼링: 최소화 (99% 케이스에서 끊김 없음)
- 동시 접속: 500명 (피크 타임 기준)
- 업타임: 99.5%

## 9. 경쟁사 분석

### 9.1 직접 경쟁사

**HireVue** (글로벌 1위)

- 강점: AI 분석 고도화, 대기업 레퍼런스 多
- 약점: 가격 비쌈 ($$$), 한국 시장 약함
- 차별점: 우리는 원티드 플랫폼 내 자연스런 통합

**Myinterview** (호주)

- 강점: 사용자 친화적 UI, 중소기업 타겟
- 약점: 한국어 지원 없음
- 차별점: 우리는 한국 채용 문화 이해

### 9.2 간접 경쟁사

- 프로그래머스 (코딩 테스트): 개발자 특화, 면접은 없음
- 리멤버 커리어: 추천 기반, 사전 면접 기능 無

### 9.3 우리의 차별점

1. **원티드 생태계 통합**: 공고-지원-면접이 한 곳에서
2. **진입 장벽 낮음**: 무료 플랜, 쉬운 설정
3. **한국형 UX**: 한국 채용 프로세스에 최적화
4. **양방향 가치**: 기업도, 지원자도 모두 이득

## 10. 리스크 및 대응 방안

### 10.1 기술 리스크

**R-01: 브라우저 호환성 이슈**

- 리스크: 특정 브라우저/OS에서 카메라 작동 안 됨
- 확률: Medium
- 영향: High
- 대응: 지원 브라우저 명확히 안내 (Chrome, Safari 최신버전)
- 완화: 녹화 전 테스트 단계 필수 도입

**R-02: 영상 업로드 실패**

- 리스크: 네트워크 불안정, 파일 크기 초과
- 확률: High
- 영향: Critical
- 대응: 재시도 로직, 압축 알고리즘, 진행률 표시
- 완화: 로컬 임시 저장 후 백그라운드 업로드

### 10.2 비즈니스 리스크

**R-03: 지원자 참여율 저조**

- 리스크: "귀찮다", "부담스럽다" 느끼면 사용 안 함
- 확률: High
- 영향: Critical
- 대응:
    - 혜택 명확화 ("합격률 2배 상승" 등 데이터)
    - 인센티브 (면접 완료 시 원티드 포인트 지급)
    - 옵셔널하게 유지 (강제 아님)

**R-04: 채용담당자 사용 미흡**

- 리스크: 영상 보는 시간 없다, 귀찮다
- 확률: Medium
- 영향: High
- 대응:
    - 하이라이트 기능 (AI가 핵심 부분만 추출)
    - 배속 재생, 모바일 최적화
    - 온보딩 웨비나 제공

### 10.3 법적/윤리 리스크

**R-05: 개인정보 및 영상 저장 이슈**

- 리스크: GDPR, 개인정보보호법 위반 소지
- 확률: Medium
- 영향: Critical
- 대응:
    - 법무팀 검토 필수
    - 명확한 동의 절차
    - 영상 암호화 저장
    - 보관 기간 명시 (최대 1년)

**R-06: AI 편향성 논란**

- 리스크: "AI가 특정 인종/성별 차별한다" 논란
- 확률: Low (Phase 1에서는 AI 분석 없음)
- 영향: Critical (브랜드 이미지)
- 대응:
    - Phase 1에서는 AI 분석 제외
    - Phase 2 진입 시 편향성 테스트 필수
    - 투명성 확보 (AI 평가 기준 공개)

## 15. 오픈 이슈 / 미해결 질문

> 리뷰 때 함께 논의하고 싶은 부분들입니다!

**Q1. 사전면접을 "필수"로 해야 할까요, "선택"으로 해야 할까요?**

- 필수로 하면 채용 품질은 올라가지만, 지원자 이탈 우려
- 선택으로 하면 참여율이 너무 낮을 수도...
- 제 생각: 일단 "선택"으로 시작, 데이터 보고 판단

**Q2. 재시도를 몇 번까지 허용해야 할까요?**

- 무제한 허용 → 진정성 의심
- 1회만 허용 → 긴장해서 망치면 억울함
- 제 생각: 채용담당자가 설정하도록

**Q3. AI 피드백을 Phase 1에 넣어야 할까요?**

- 개발팀은 "너무 촉박하다"
- 하지만 지원자 입장에서는 이게 제일 궁금할 듯
- 제 생각: Phase 2로 밀되, 베타 기간에 수동으로라도 피드백 제공

**Q4. 영상 화질을 어디까지 허용해야 할까요?**

- 720p: 적당, 파일 크기 적정
- 1080p: 선명하지만 업로드 시간 증가
- 480p: 빠르지만 너무 흐림
- 제 생각: 기본 720p, 네트워크 상황 따라 자동 조절

**Q5. 가격 정책이 적절한가요?**

- 월 49,000원이 비싼지 싼지 감이 안 잡힘
- 경쟁사 HireVue는 연 수백만원 수준
- 제 생각: 베타 기간 무료 → 피드백 받고 가격 재조정
